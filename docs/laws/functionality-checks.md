# Functionality-check register — every sprint's real-environment proof

Every sprint ends with a **functionality check**: the delivered capability exercised in a *realistic*
environment (not just unit tests — **LAW-02**, success is proven, never assumed). `make ci` green is
necessary but not sufficient. Each entry names what was run, against which **real** systems, the result
(with evidence), and confirms **teardown** — every test artifact removed, the environment left as found.

**Procedure (per sprint):** run the check → tear down (delete stamped graph rows, cancel paper orders,
delete scratch files) → append a row here. If a sprint's feature has no realistic target yet (a
primitive awaiting later sprints), say so and check the nearest real thing it enables.

Dev graph store for checks: **Neon PostgreSQL via `POSTGRES_DSN`**. Return stamped rows to zero after
each check. Historical Aura rows below remain valid evidence for the pre-S117/S118 Neo4j spine.

**Standing broker state note (DL-44 / S120):** `Position` and `Fill` rows that mirror real Alpaca
paper holdings are production state, not disposable proof artifacts. Checks that trade must run
broker reconciliation after teardown so the graph mirrors the standing account again. Stamped test
extras are still fully torn down to zero.
